# **瞎说的建议**

---

> - 准备好一款在线、随时随地写代码的工具，不用在本地去安装和配置，是初学者的好帮手。
> - 准备一款记笔记的软件，学编程的过程中老师的思路、自己遇到的问题、解决问题的方法、心得感悟、遇到的好资料，都要记下来哦，好记性不如烂笔头，可以帮助你事半功倍。
> - 想学好编程，别背代码，要多写，把这句话刻到脑子里。
> - 选视频和找资料的小技巧：多看评论区、多看隔壁评论区、不要看过期的视频。

# **心得体会**

---

> [!TIP]
>
> - 总结自己的一套学习方法至关重要，甚至影响你程序员的一生，望大家重视起来

> - 听课（保证考勤）									//当天
> - 练习（操作演练）									//当天
> - 总结（归纳重点，查漏补缺）								//当天/次日
> - 扩展（积累经验，收集素材）								//本周
> - 分享（学习就像盲人摸象，分享就是跟别人说，找问题，沉淀知识）				//本周/本月

> - 知道自己在学什么，明确自己的学习目标
> - 建立自己的知识网络，比较简单的方法就是善用思维导图，把知识网络串起来
> - 从大量信息中筛选有价值信息的能力，人的精力是有限的，我们不可能面面俱到，学习一定要学经典资料
